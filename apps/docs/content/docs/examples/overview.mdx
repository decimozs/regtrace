The [examples/](https://github.com/decimozs/regtrace/tree/main/examples) directory
contains ready-to-run Regtrace projects covering real-world production scenarios.
Each example is self-contained — its own golden sets, config, and README.

## How to run

Pick an example and run it:

```bash
cd examples/<name>
regtrace run
```

No API key needed for format-only examples — they use deterministic metrics.
For full factuality and tone evaluation, add an LLM judge key (`.env`) and
enable the relevant metrics in `regtrace.config.yaml`.

## Feature coverage

Each example exercises different Regtrace features:

- **Quality gates** — all examples configure `suite_score_minimum`,
  `max_failed_test_cases`, and `regression_gate`
- **Regression tracking** — every run stores a baseline for delta comparison
- **Format sub-checks** — length, JSON validity, JSON schema, markdown structure,
  required fields, forbidden content
- **Deterministic by default** — all examples default to format-only so you can
  run them immediately
- **RAG interaction type** — three RAG examples demonstrate
  `context.documents` with retrieval scores and faithfulness evaluation
